# EQ 改作業小幫手：Google Cloud OAuth2 設定指南

本指南將引導您從零開始設定 Google 登入功能，讓老師們可以安全地進入管理後台。

## 1. 前置準備

在開始之前，請確認您具備以下條件：
*   一個有效的 Google 帳號（個人 Gmail 或學校 Workspace 帳號皆可）。
*   電腦上已安裝 Chrome 或 Safari 等網頁瀏覽器。
*   可以編輯電腦中檔案的權限。

## 2. 建立 Google Cloud 專案

1.  開啟瀏覽器，前往 [Google Cloud Console](https://console.cloud.google.com/)。
2.  如果您是第一次使用，請勾選同意服務條款，然後點擊「同意並繼續」。
3.  在頁面頂端，點擊「選取專案」（或是目前的專案名稱），然後在彈出的視窗右上角點擊「新專案」。
4.  在「專案名稱」欄位輸入 `EQ-Grader-Project`。
5.  「位置」欄位通常維持預設即可（若您有組織帳號，請選擇您的學校組織）。
6.  點擊「建立」。系統需要幾秒鐘的時間處理，完成後請確認頂端的專案選單已切換至剛建立的專案。

## 3. 設定 OAuth 同意畫面

這是使用者在登入時會看到的授權畫面。

1.  點擊左上角的選單圖示（三條橫線），選擇「API 和服務」 > 「OAuth 同意畫面」。
2.  在「User Type」選擇「外部 (External)」，然後點擊「建立」。
3.  填寫「應用程式資訊」：
    *   **應用程式名稱**：輸入 `EQ改作業小幫手`。
    *   **使用者支援電子郵件**：從下拉選單選擇您的電子郵件。
    *   **應用程式標誌**：可不上傳。
4.  填寫「開發人員聯絡資訊」：
    *   **電子郵件地址**：輸入您的電子郵件。
5.  點擊「儲存並繼續」。
6.  進入「範圍 (Scopes)」設定頁面，點擊「新增或移除範圍」。
7.  在彈出的視窗中，勾選前三個選項：
    *   `.../auth/userinfo.email`
    *   `.../auth/userinfo.profile`
    *   `openid`
8.  捲動到視窗底部，點擊「更新」。
9.  回到頁面底部，點擊「儲存並繼續」。
10. 進入「測試使用者 (Test users)」頁面，點擊「ADD USERS」。
11. 輸入您自己以及其他需要登入系統的老師電子郵件地址。每輸入一個請按一次 Enter。
12. 點擊「增加」，然後點擊「儲存並繼續」。
13. 最後確認摘要資訊無誤後，點擊「返回資訊主頁」。

## 4. 建立 OAuth 憑證

1.  在左側選單點擊「憑證」。
2.  點擊頂端的「建立憑證」，選擇「OAuth 用戶端 ID」。
3.  「應用程式類型」選擇「網頁應用程式」。
4.  「名稱」輸入 `EQ-Grader-Web-Client`。
5.  找到「已授權的重新導向 URI」區塊，點擊「新增 URI」。
6.  輸入以下兩個網址（請務必完全一致）：
    *   `http://localhost:8000/auth/callback`
    *   `http://192.168.x.x:8000/auth/callback` (請將 `192.168.x.x` 替換成您 Mac Mini 的實際區域網路 IP 地址)
7.  點擊「建立」。
8.  畫面會彈出「OAuth 用戶端已建立」視窗，顯示您的「用戶端 ID」和「用戶端密鑰」。請先不要關閉此視窗，或將這兩串字串複製下來備用。

## 5. 填入設定

現在需要將取得的資訊填入應用程式的設定檔中。

1.  在您的 Mac Mini 上，找到資料夾 `/Users/michael/Desktop/ActiveKM/eq-grader/`。
2.  尋找名為 `.env` 的檔案並開啟它。
3.  找到以下兩行，刪除前方的 `#` 符號（如果有），並填入剛才取得的資訊：
    ```
    GOOGLE_CLIENT_ID=您的用戶端ID.apps.googleusercontent.com
    GOOGLE_CLIENT_SECRET=您的用戶端密鑰
    ```
4.  儲存檔案並關閉。
5.  **重要**：修改完設定後，必須重新啟動「EQ改作業小幫手」程式，新設定才會生效。

## 6. 驗證

1.  開啟瀏覽器，前往 `http://localhost:8000/teacher`。
2.  您應該會看到「使用 Google 帳號登入」的按鈕。
3.  點擊按鈕，選擇您在「測試使用者」步驟中加入的電子郵件。
4.  登入成功後，系統應會導向老師管理後台。

## 7. 正式上線注意事項

*   **發布應用程式**：目前應用程式處於「測試」模式，只有被加入「測試使用者」名單的人可以登入。若要讓更多老師使用，您可以回到「OAuth 同意畫面」點擊「發布應用程式」。
*   **新增重新導向網址**：如果您更換了網路環境導致 Mac Mini 的 IP 地址改變，請記得回到 Google Cloud Console 的「憑證」頁面，修改「已授權的重新導向 URI」。

## 8. 常見問題

*   **出現 "redirect_uri_mismatch" 錯誤**：
    這代表您在瀏覽器輸入的網址與 Google Cloud Console 設定的「已授權的重新導向 URI」不符。請檢查兩邊的網址是否完全一致，包含 `http` 還是 `https` 以及連接埠號碼 `:8000`。
*   **出現 "access_denied" 錯誤**：
    這通常是因為登入的帳號沒有被加入「測試使用者」名單中。請回到「OAuth 同意畫面」確認該電子郵件是否已正確新增。
*   **登入按鈕沒有出現**：
    請檢查 `.env` 檔案中的 `GOOGLE_CLIENT_ID` 和 `GOOGLE_CLIENT_SECRET` 是否填寫正確，且該行開頭沒有 `#` 符號。修改後請務必重啟程式。
*   **如何增加更多授權老師**：
    在應用程式尚未正式發布前，請在「OAuth 同意畫面」的「測試使用者」區塊點擊「編輯」或「新增使用者」來加入其他老師的電子郵件。
